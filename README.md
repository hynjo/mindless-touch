# mindless-touch

소리만 듣고 화면을 탭해 숨겨진 고양이 눈을 찾는 모바일 웹 프로토타입입니다. 고양이 눈은 이미지 없이 CSS로 렌더링합니다.

## 실행

Node.js 26.5.0을 사용합니다. 처음 한 번 의존성을 설치한 뒤 Vite 개발 서버를 실행하세요.

```bash
npm install
npm run dev
```

터미널에 표시된 로컬 주소를 열고 Chrome DevTools의 Device Mode를 켭니다.

## 플레이 방법

1. 빈 화면을 처음 탭하면 오디오가 활성화되고 문제 소리가 납니다.
2. 화면을 탭해 숨겨진 도형을 찾습니다. 오답이면 낮은 소리가 납니다.
3. 정답 영역을 한 번 탭하면 찾음 소리가 나지만 도형은 계속 숨겨져 있습니다.
4. 시간제한 없이 같은 숨겨진 영역을 다시 탭하면 공개 소리와 함께 도형이 표시됩니다.
5. 공개 소리가 끝난 후 화면 아무 곳이나 탭하면 정답 소리가 납니다.
6. 정답 소리가 끝나면 다음 문제가 자동으로 시작됩니다.

찾음 소리 이후 다른 영역을 터치하면 오답 소리와 함께 찾음 상태가 초기화됩니다. 다음 정답 터치는 다시 첫 번째 찾음으로 처리됩니다.

드래그는 판정하지 않습니다. 빠른 연속 탭과 멀티터치는 각 터치별로 판정합니다.

## 레벨

`level` URL 파라미터로 정답 영역의 크기를 조절합니다. `1`이 가장 쉽고 `10`이 가장 어려우며, 지정하지 않으면 `5`입니다.

같은 레벨에서는 모든 문제의 눈 크기가 동일하며 위치만 무작위로 변경됩니다.

- `?level=1`: 화면 면적의 약 9~18%
- `?level=5`: 화면 면적의 약 3.7~7.4%
- `?level=10`: 화면 면적의 약 1.2~2.4%

범위를 벗어난 숫자는 `1~10` 사이로 보정하고, 숫자가 아니면 기본값 `5`를 사용합니다.

예시:

```text
http://localhost:5173/mindless-touch/?level=8
```

## 고양이 눈 색상

`styles.css`의 CSS 변수로 공개되는 고양이의 눈 색상을 변경할 수 있습니다.

```css
:root {
  --cat-eye-color: #fff875;
}
```

고양이 눈의 판정 영역은 화면 바깥쪽 10% 안전 구역을 제외한 내부에만 배치됩니다.
실제 판정 영역은 눈 한 쌍 너비의 1.2배 지름을 가진 보이지 않는 원형 히트박스입니다. 디버그 모드에서는 점선 원으로 확인할 수 있습니다.

## 사운드 에셋

`reveal` 피드백은 `public/assets/reveal.wav`, 오답 피드백은 `public/assets/wrong.wav`, `found` 피드백은 `public/assets/found.wav`, 완료 피드백은 `public/assets/correct.wav` 파일을 사용합니다. 문제 시작음은 브라우저에서 합성합니다. 라운드를 완료하면 고양이 눈이 잠시 커지고 밝아지며, 완료음이 끝난 뒤 200ms 후 다음 라운드가 시작됩니다.

## 디버그

- `?debug` 또는 `?debug=1`: 문제를 푸는 동안에도 정답 도형을 표시하고 정답·오답 터치 위치를 점으로 표시
- `?seed=example`: 새로고침해도 같은 문제 순서 재현
- 함께 사용: `?debug=1&seed=example&level=8`

## GitHub Pages

프로덕션 파일을 생성합니다.

```bash
npm run build
```

결과물은 `dist/`에 생성됩니다. Vite의 `base`는 GitHub Pages의 `/mindless-touch/` 경로로 설정되어 있습니다. GitHub Pages에는 `dist/` 결과물을 배포하면 됩니다.
